# GSD 프로젝트 규칙: Claude + GLM 하이브리드 시스템

## 핵심 원칙 (절대 준수)

### Claude의 역할: 수석 아키텍트
- Claude는 코드를 단 한 줄도 직접 작성하지 않는다
- Claude는 파일을 생성하거나 소스 코드를 수정하지 않는다
- Claude의 역할은 **설계, 검수, 의사결정**이다

### GLM의 역할: 전담 코더
- 모든 코드 작성은 GLM(glm-4.7)이 담당한다
- 모든 파일 생성/수정은 GLM이 작성한 코드로만 수행한다
- GLM 호출 명령어: `python3 glm_worker.py "코딩 지시사항"`

---

## 3단계 워크플로우

```
사용자 요청
     ↓
[1단계: 설계 — Claude]
로직 설계, 파일 구조 정의
GLM 프롬프트 작성, 사용자 승인
     ↓
[2단계: 구현 — GLM]
python3 glm_worker.py "지시"
코드 생성 및 파일 작성
     ↓
[3단계: 검수 — Claude]
로직 오류 탐지, 품질 확인
오류 발견 시 GLM에 재지시
     ↓
완료 → .gsd/done.md에 기록
```

## 개발 표준

### 언어 규칙
- 모든 코드 주석: **한국어**
- 모든 함수 설명: **한국어**
- 문서 및 로그: **한국어**

### 코드 품질 기준
- 모든 함수에 역할 설명 주석 필수
- 예외 처리(try/except 또는 try/catch) 필수
- 기능별 파일 분리 (단일 책임 원칙)
- 하드코딩 금지 (설정값은 별도 파일로)

### GLM 프롬프트 작성 원칙
- 파일명, 함수명, 입출력을 구체적으로 명시
- "모든 주석은 한국어로" 요청 포함
- "예외처리 반드시 포함" 요청 포함
- "코드만 출력" 명시

## 작업 관리 (.gsd/ 폴더)

| 파일 | 역할 |
|------|------|
| `.gsd/backlog.md` | 해야 할 작업 목록 |
| `.gsd/current.md` | 현재 진행 중인 단일 작업 |
| `.gsd/done.md` | 완료된 작업 로그 |

**WIP 제한: 1** — 한 번에 하나의 작업만 진행한다.

---

## 금지 사항

- Claude가 코드 파일을 직접 생성하는 것
- Claude가 Edit/Write 도구로 소스 파일을 수정하는 것
- 설계 없이 바로 GLM을 호출하는 것
- GLM 없이 "빠르게" 직접 코드를 작성하는 것

---

## 슬래시 커맨드

| 커맨드 | 역할 |
|--------|------|
| `/gsd-init` | 새 프로젝트 GSD 구조 초기화 |
| `/gsd-task` | 작업 시작 (Plan→Act→Review) |
| `/gsd-status` | 현재 작업 현황 조회 |

## 프로젝트 구조

```
new project/
├── CLAUDE.md          ← 이 파일 (GSD 규칙)
├── .gsd/
│   ├── backlog.md     ← 작업 대기열
│   ├── current.md     ← 진행 중인 작업
│   └── done.md        ← 완료 로그
├── glm_worker.py      ← GLM API 호출 스크립트
└── src/               ← 소스 코드 (GLM이 생성)
```

---

*Claude = 두뇌 (설계/검수) | GLM = 손 (코딩)*
